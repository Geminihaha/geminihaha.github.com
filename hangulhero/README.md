# 프로젝트 기획서: 한글배우기(HangulQuest) / hangulhero

이 문서는 7세 미만 아동을 위한 한글 교육용 웹 애플리케이션 '한글배우기(HangulQuest)'의 기획서입니다.

---

## 1. 공통 가이드라인 (System Instruction)
* **타겟:** 만 7세 이하 아동
* **스타일:** 파스텔 톤, 둥근 모서리(Rounded), 친근하고 직관적인 UI
* **접근성:** 버튼은 크게(최소 48x48px), 텍스트는 최소화하고 아이콘 중심 설계
* **피드백:** 즉각적이고 긍정적인 반응(애니메이션, 효과음 암시)

---

## 2. 화면별 구성 및 디자인 사양

### A. 메인 허브 (학습 지도 화면)
* **컨셉:** 아이들이 탐험하는 느낌을 주는 '학습 지도' 메인 화면 (기본 14개 전체 자음 지원)
* **배경:** 부드러운 언덕과 숲이 있는 모험 테마
* **구성:** 학습 단계별로 연결된 '길'과 그 위에 배치된 '14개의 자음 스테이지 아이콘(별 모양)'
* **특징:** 완료된 스테이지는 밝게 빛나고, 잠긴 스테이지는 자물쇠 아이콘이 표시됨. 전체 14개 자음을 세로로 스크롤하여 탐험할 수 있는 맵 설계.
* **분위기:** 알록달록하지만 눈이 편안한 파스텔 톤

### B. 학습 모드 (글자 쓰기 화면)
* **컨셉:** 아이들이 한글 자모음을 따라 쓰는 학습 화면
* **상단:** 뒤로가기 버튼, 현재 글자(예: 'ㄱ'), 점수 아이콘(별/보석 개수)
* **중앙:** 화면 가득 찬 크기의 옅은 회색 가이드라인 글자(ㄱ). 사용자가 터치/드래그로 잉크를 채울 수 있는 캔버스 영역
* **하단:** '다시 쓰기(화살표)', '힌트(전구)', '다음(화살표)' 버튼
* **인터랙션:** 글자를 쓰면 잉크가 밝게 빛나는 느낌의 스타일 강조

### C. 퀴즈 모드 (드래그 앤 드롭)
* **컨셉:** 재미있게 복습하는 한글 매칭 퀴즈 게임 화면
* **중앙:** 제시된 그림(사과) 1개
* **하단:** 선택 가능한 글자 블록 3개(가, 나, 사)
* **인터랙션:** 아이가 글자를 그림으로 드래그하면 쏙 들어가는 배치
* **피드백:** 정답 시 캐릭터가 춤을 추는 팝업 애니메이션 영역 확보

### D. 보상 창 (성과 확인 화면)
* **컨셉:** 학습 완료 후 뜨는 축하 보상 팝업 화면
* **구성:** '학습 완료!' 축하 문구, 획득한 보석 3개, 스티커 북 버튼
* **분위기:** 폭죽이 터지는 듯한 화려하면서도 귀여운 애니메이션 효과 강조
* **목적:** 아이에게 성취감을 주는 밝고 에너지 넘치는 디자인

---

## 3. 작업 시 유의사항
1. **반응형 체크:** 태블릿 및 모바일 기기(세로 모드) 환경에서의 버튼 위치와 터치 영역 고려
2. **일관성:** 모든 화면의 버튼 디자인과 컬러 팔레트가 동일하게 유지되도록 'Design System' 정의
3. **디자인 레퍼런스:** `design/stitch_game_redesign_project`의 고품질 테마와 컴포넌트 스타일을 참고하여 한글 학습용 파스텔 테마로 변형 적용
