# Queens Logic Puzzle Game - Project Specification

## 1. 프로젝트 개요 (Project Overview)
본 프로젝트는 N-Queen 문제의 논리적 규칙에 '색상 구역(Region) 제한'을 결합한 두뇌 논리 퍼즐 게임 애플리케이션입니다. 

## 2. 기술 스택 및 아키텍처 (Tech Stack & Architecture)
* **언어:** Kotlin
* **아키텍처:** MVVM (Model-View-ViewModel) 패턴
* **화면 전환:** Fragment 기반의 Navigation
* **UI 바인딩:** ViewBinding
* **로컬 데이터베이스:** Room DB (수백 개의 스테이지 맵 데이터, 색상 구역 배열, 유저 클리어 타임 기록 저장)

## 3. 핵심 퍼즐 로직 (Core Puzzle Logic)
에이전트는 다음 세 가지 규칙을 완벽하게 검증하는 로직을 구현해야 합니다.
1.  **가로/세로 제한:** 보드의 각 행(Row)과 열(Column)에는 단 하나의 퀸만 존재해야 함.
2.  **색상 구역 제한:** 불규칙하게 나뉜 각 색상 구역(Region) 내에도 단 하나의 퀸만 존재해야 함.
3.  **인접 제한:** 퀸끼리는 상하좌우 및 대각선으로 바로 인접한 칸(8방향)에 놓일 수 없음.
* **난이도 구성:** 5x5(초급), 7x7(중급), 9x9(고급) 보드로 확장 가능하도록 상태 배열을 유연하게 설계할 것.

## 4. UI/UX 및 상태 관리 (UI/UX & State Management)
* **뷰 모델 상태 유지:** 화면 회전 또는 앱 백그라운드 전환 시에도 보드의 퀸 배치 상태와 'X(절대 불가 메모)' 마크 상태가 유실되지 않도록 `ViewModel`을 통해 보드 상태 배열을 관리.
* **UI 렌더링 최적화:** 보드의 특정 칸을 터치하여 퀸이나 X 마크를 배치/취소할 때, 화면 전체가 아닌 해당 칸의 UI만 독립적으로 갱신되도록 ViewBinding과 재사용 가능한 뷰 구조를 활용해 렌더링 부하를 최소화.
* **시각적 피드백:** * 규칙 위반(충돌) 시, 해당 퀸과 줄/구역을 붉은색으로 깜빡이는(Blink) 효과 적용.
    * 플랫 & 파스텔 톤 기반의 UI 구성으로 눈의 피로도 최소화.

## 5. 에이전트 작업 지시 (Agent Action Items)
Antigravity CLI 에이전트는 위 명세를 바탕으로 다음 작업을 순차적으로 수행할 것:
1.  **프로젝트 스캐폴딩:** 위 기술 스택에 맞춘 프로젝트 기초 구조 및 디렉토리 세팅.
2.  **Room DB 스키마 작성:** 스테이지 정보를 담을 Entity와 DAO 구현.
3.  **상태 관리 객체 구현:** 퍼즐의 3가지 코어 규칙을 실시간으로 검증하고 에러를 반환하는 ViewModel 및 로직 클래스 작성.
4.  **UI 컴포넌트 개발:** ViewBinding을 활용하여 터치 상호작용(1번 터치: 퀸, 길게/더블 터치: X 마크)이 가능한 보드판 뷰 및 Fragment 뷰단 구현.
