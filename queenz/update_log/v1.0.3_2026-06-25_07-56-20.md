## 📅 주요 업데이트 완료 내역

### 1. 실시간 버전 체크 및 하드 새로고침 자동화 (Cache Busting)
브라우저의 강력한 정적 에셋 캐싱으로 인해 앱 업데이트가 화면에 즉시 반영되지 않는 고질적인 웹 캐시 문제를 해결했습니다.
* **서버 레벨 캐시 방지**: [server.js](file:///data/data/com.termux/files/home/workspace/geminihaha.github.com/queenz/server.js)의 응답 헤더에 `Cache-Control: no-store, no-cache, must-revalidate, max-age=0` 정책을 적용해, 브라우저가 매번 서버의 실제 코드를 조회하도록 조치했습니다.
* **클라이언트 버전 쿼리**: [index.html](file:///data/data/com.termux/files/home/workspace/geminihaha.github.com/queenz/index.html)에서 로드하는 스타일 및 스크립트 경로에 `?v=1.0.3` 버전 파라미터를 강제 적용하여 리소스를 항상 최신 상태로 강제 교체합니다.
* **버전 미스매치 시 하드 리로드**: [app.js](file:///data/data/com.termux/files/home/workspace/geminihaha.github.com/queenz/app.js)에 전역 버전 변수(`APP_VERSION`)를 심고, 유저 로컬스토리지의 기록과 불일치가 발견되면 즉각 최신 버전 정보를 갱신한 뒤 브라우저 하드 리로드(`window.location.reload(true)`)를 1회 자동 실행하여 코드 갱신을 보장합니다.

### 2. 기록 데이터 백업(Backup) 및 병합 복원(Restore) 기능 구현
유저의 소중한 퍼즐 클리어 기록(`queens_puzzle_records_v1` LocalStorage 키)을 기기에 저장하고 타 브라우저나 기기에서 복원할 수 있도록 입출력 시스템을 구축했습니다.
* **UI/UX 디자인 통합**: 메인 로비 통계판 바로 하단에 심플하고 세련된 글래스모피즘 아크릴 스타일의 **[📥 기록 백업]** 및 **[📤 기록 복원]** 버튼 디자인 추가 및 배치 완료.
* **기록 백업 (JSON 다운로드)**: 로컬 저장소의 데이터를 정교하게 직렬화하여 날짜 포맷이 붙은 `queens_puzzle_backup_YYYY-MM-DD.json` 파일 형태로 브라우저 다운로드를 강제 실행합니다.
* **기록 복원 및 병합 (JSON 업로드)**: 
  * 파일 선택기를 통해 json을 업로드하면 `FileReader`로 비동기식 덤프를 파싱합니다.
  * 데이터 포맷과 스키마를 유효성 검사하여 파손된 파일 주입을 사전 차단합니다.
  * **병합 복원(Smart Merge) 지원**: 가져온 기록 중 **기존보다 더 빠른 클리어 시간만** 스마트하게 판별하여 선별적으로 로컬 레코드와 합칩니다. 이를 통해 유저가 기존에 세워둔 신기록이 복원 과정에서 덮어씌워져 유실되지 않도록 완벽히 보존합니다.
