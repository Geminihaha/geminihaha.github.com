# [기획서] 아이들을 위한 반응형 색칠놀이 웹앱 (Coloring Sandbox)

본 기획서는 모바일 및 태블릿 환경에 최적화된 아동용 반응형 색칠놀이 웹앱의 요구사항 및 개발 명세를 정의합니다. 아동이 사용하는 만큼 직관적인 UI/UX와 터치 인터페이스 최적화에 중점을 둡니다.

---

## 1. 프로젝트 개요
*   **목적**: 아이들이 모바일/태블릿에서 자유롭게 그림을 그리고, 이미지를 가져와 색칠놀이를 즐길 수 있는 웹 기반 애플리케이션
*   **주요 타깃**: 5세 ~ 10세 아동 및 부모
*   **핵심 가치**: 직관성(텍스트 최소화, 아이콘 중심), 반응성(다양한 스크린 크기 지원), 재미(다양한 브러시 질감)
*   **주요 기술 스택 (추천)**: HTML5 Canvas API (또는 Fabric.js / Pixi.js), React 또는 Vanilla JS, Tailwind CSS (반응형 UI)

---

## 2. 핵심 기능 정의

### 2.1. 캔버스 영역 (Drawing Canvas)
*   **자유 드로잉**: 멀티터치 및 단일 터치 입력을 감지하여 실시간 선 구현 (터치 지연 최소화 `touch-action: none`).
*   **반응형 스크린**: 기기 방향(가로/세로) 및 해상도에 맞춰 캔버스 크기가 유연하게 조절됨.
*   **배경 레이어 기능**: 사용자가 가져온 이미지(도안)를 하단 레이어에 배치하고, 그 위에 투명한 드로잉 레이어를 두어 원본 훼손 없이 색칠할 수 있도록 구성.

### 2.2. 브러시 툴박스 (Brush Toolbox)
요청하신 3가지 타입의 펜 질감을 Canvas의 `globalCompositeOperation`, `shadow`, 또는 패턴 비트맵을 활용해 구현합니다.
1.  **펜 (Pen)**: 깔끔하고 명확한 솔리드 선 (`lineCap: 'round'`, `lineJoin: 'round'`).
2.  **붓 (Brush)**: 불투명도가 낮고 끝이 부드럽게 퍼지는 수채화 느낌 (외곽 흐림 처리 및 압력 시뮬레이션).
3.  **크레파스 (Crayon)**: 질감이 살아있는 거친 텍스트 효과 (노이즈 패턴 텍스처 필터 적용).
*   **부가 기능**: 브러시 두께 조절 슬라이더 (직관적인 굵기 아이콘 표시).

### 2.3. 컬러 팔레트 (Color Palette)
*   아이들이 좋아할 만한 선명한 기본 12~16색 배치.
*   선택 시 시각적 피드백(크기가 커지거나 체크 표시).
*   지우개 툴 포함.

### 2.4. 외부 이미지 불러오기 (Capture & Load)
*   **카메라 촬영/캡처**: 기기 카메라를 호출하여 사진을 찍고 바로 캔버스 배경으로 삽입.
*   **갤러리 로드**: 디바이스 내 저장된 사진이나 인터넷에서 다운받은 도안 이미지를 업로드하여 배경으로 활용.
*   **도안 변환 필터 (옵션)**: 일반 사진을 올렸을 때 색칠하기 좋도록 외곽선만 추출하는 흑백 필터(Edge Detection) 적용 기능 검토.

### 2.5. 유틸리티 기능
*   **되돌리기 / 다시실행 (Undo / Redo)**: 최근 작업 10단계 저장.
*   **전체 지우기 (Clear)**: 캔버스 초기화 (실수 방지를 위해 팝업 확인창 적용).
*   **저장하기 (Save)**: 완성된 그림을 디바이스 갤러리에 이미지(PNG)로 저장.

---

## 3. UI/UX 및 와이어프레임 설계 (반응형 레이아웃)

아동용 앱이므로 모든 메뉴는 하단 또는 좌우측에 **큰 아이콘 형태**로 배치하여 손가락 터치가 쉽도록 합니다.

### 3.1. 태블릿/가로 모드 (Horizontal Layout)
*   **중앙**: 메인 캔버스 영역
*   **좌측**: 브러시 종류 (펜, 붓, 크레파스) 및 두께 조절, 지우개, 이미지 로드, 저장 버튼
*   **하단**: 컬러 팔레트 (한눈에 들어오는 가로 배열)

### 3.2. 모바일/세로 모드 (Vertical Layout)
*   **상단**: 유틸리티 메뉴 (Undo, Redo, Clear, Save, 이미지 로드)
*   **중앙**: 캔버스 영역
*   **하단 1단**: 브러시 선택 및 두께 슬라이더
*   **하단 2단**: 스크롤 가능한 컬러 팔레트

## 4. 개발 단계를 위한 요구사항 명세 (Functional Requirements)

### F-101: 캔버스 터치 이벤트 최적화
*   모바일/태블릿 브라우저에서 스크롤 및 핀치 줌이 드로잉을 방해하지 않도록 이벤트 캡처.
*   `pointerevents`를 사용하여 스타일러스 펜(예: 애플펜슬, 갤럭시 S펜)과 손가락 터치를 모두 지원.

### F-102: 브러시 질감 구현 매커니즘
*   **크레파스 구현 가이드**: 
    ```javascript
    // 기본 예시 콘셉트: 이미지를 패턴으로 사용하거나 점을 겹쳐 찍는 방식
    ctx.lineJoin = ctx.lineCap = 'round';
    ctx.strokeStyle = customCrayonPattern; // 크레파스 텍스처 매핑
    ```

### F-103: 이미지 업로드 및 Canvas 그리기
*   `<input type="file" accept="image/*" capture="environment">`를 활용해 카메라와 갤러리 동시 대응.
*   불러온 이미지는 캔버스 크기에 맞춰 비율 유지(Aspect Ratio)하며 중앙 정렬(Contain 방식).

---

## 5. 단계별 개발 로드맵

*   **1단계 (MVP - 최소 기능 제품)**: Canvas 기본 드로잉 + 기본 펜 기능 + 기본 컬러 팔레트 + 모바일 터치 대응.
*   **2단계 (질감 및 유틸리티)**: 붓/크레파스 질감 구현 + Undo/Redo 기능 + 이미지 저장 및 불러오기 기능 추가.
*   **3단계 (UI/UX 고도화)**: 사운드 효과(슥슥 소리, 버튼 클릭음), 사진 외곽선 변환 필터 적용, 아동 친화적 애니메이션 추가.
