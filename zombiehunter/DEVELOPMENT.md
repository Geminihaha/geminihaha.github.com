# Zombie Hunter — 개발 기록

## 개요

뱀파이어 서바이버 스타일의 웹 게임. 순수 HTML + Canvas + JavaScript로 제작됨.

- **원본 출처**: Ghost Hero 레포지토리 내 prototype
- **플레이**: `zombiehunter/index.html` 브라우저 열기
- **빌드 없음**: 정적 HTML+JS, 번들러 불필요

## 수정 내역

### 2026-07-07 — 채찍(Whip) 공격 히트 판정 버그 수정

#### 버그 증상
1번 캐릭터(안토니오, 채찍) 자동 공격 시 범위 호(arc)는 시각적으로 표시되나, 호 안에 몬스터가 들어가도 데미지를 전혀 주지 못함.

#### 원인 분석
`game.js`의 `WEAPON_DEFS.whip.update()` 내 `hit:` 함수에서 거리 조건(`dist < r`)과 각도 조건(`Math.abs(d) < arc/2+0.2`)이 있었으나, 각도 계산(`d = a - angle`, `angle`은 공격 생성 시점의 플레이어 방향)과 시각적 호(`ctx.arc(0,0,r,-arc/2,arc/2)`) 간 정합성 문제로 호 안의 적이 피격되지 않음.

#### 수정 사항
- **히트 함수 재작성** (`game.js:49-58`): 거리(90px) 조기 탈출 + 각도 차이 정규화(`[-PI, PI]`) 후 `arc/2+0.2` 이내 확인
- **진단 도구 추가 후 제거**:
  - `hit()` → `return true` 무조건 명중 모드로 파이프라인 정상 확인
  - `g.screenshake=12` 화면 흔들림으로 명중 시각화
  - 히트 카운터 HUD 표시 (`히트:N`, 우측 하단 디버그 영역)
  - 데미지 팝업 시각 강화 (22px 볼드 + 검은 테두리)
- **arc/r 변수 선언 보호**: 이전 리팩토링에서 `console.log` 제거 시 `const arc=PI/2, r=90*g.areaMuL` 변수 선언이 함께 삭제되어 `ReferenceError` 발생 → 복구

#### 검증
- Android 15 Termux Chrome 모바일 환경에서 정상 작동 확인
- `node --check game.js` 문법 검증 통과
- 모든 캐릭터(마법봉 투사체, 단검, 도끼 등) 무기 발사 및 피격 정상
