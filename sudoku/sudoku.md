# Sudoku Game 개발 프로젝트 리포트

EaselJS 라이브러리를 기반으로 한 웹 브라우저용 수도쿠 게임 개발 프로젝트입니다.

## 1. 프로젝트 개요
- **기술 스택**: HTML5, CSS3, JavaScript (EaselJS 0.8.2)
- **주요 목표**: 모바일 및 데스크탑 환경을 모두 지원하며, 다양한 난이도와 퍼즐 번호 선택 기능을 갖춘 반응형 수도쿠 게임 구현.

## 2. 핵심 구현 기능

### 2.1. 게임 로직 및 퍼즐 생성
- **다양한 그리드 지원**: 9x9 표준 수도쿠와 6x6(가로 3, 세로 2 블록) 미니 수도쿠 지원.
- **정답판 기반 퍼즐 생성 (Solved Board-based Generation)**:
    - 불완전한 패턴을 수정하는 방식 대신, 수학적으로 완벽하게 검증된 정답판(`solvedBases`)을 기본 데이터로 사용.
    - 이를 통해 6x6 등의 소형 그리드에서 발생할 수 있는 논리적 오류(특정 숫자가 들어갈 수 없는 현상)를 근본적으로 차단.
- **Hole Digging 알고리즘**:
    - 완성된 정답판에서 난이도(초급/중급/고수)에 따라 무작위로 숫자를 제거하여 퍼즐을 생성.
    - 이 방식은 생성된 모든 퍼즐이 반드시 하나 이상의 정답을 가짐을 보장함.
- **시드 기반 셔플 알고리즘 (Fisher-Yates Shuffle 기반)**: 
    - **알고리즘 명**: 피셔-예이츠 셔플(Fisher-Yates Shuffle) 알고리즘을 시드 기반 난수 생성기와 결합하여 구현.
    - **도입 이유**:
        1. **재현성(Reproducibility)**: 동일한 퍼즐 번호(시드)를 입력했을 때 전 세계 어디서든 항상 동일한 퍼즐 배열을 생성하기 위함.
        2. **데이터 효율성**: 수백 개의 퍼즐 배열을 직접 텍스트로 저장하는 대신, 하나의 기본 패턴과 시드값만으로 수천 개의 유효한 퍼즐을 즉석에서 생성하여 저장 공간을 혁신적으로 절약(약 16.5KB 내외).
        3. **무결성 보장**: 단순 무작위 섞기가 아닌, 수도쿠의 규칙(행, 열, 블록 제약)을 유지하는 범위 내에서 셔플을 수행하여 항상 풀 수 있는 유효한 퍼즐임을 보장.
    - **구현 방식**: 기본 퍼즐 패턴을 바탕으로 숫자 치환(Symbol Mapping), 행/열 블록 내 셔플(Block Shuffling)을 수행.
- **난이도 시스템**: 초급, 중급, 고수 3단계 난이도 제공.

### 2.2. 사용자 인터페이스(UI) 및 UX
- **반응형 디자인**: Canvas 크기가 화면 너비에 맞게 조절되며, 상단 정렬 레이아웃 적용.
- **모바일 최적화 키패드**: 모바일 사용자를 위해 하단에 숫자 선택 버튼 및 'clear' 버튼 배치.
- **실시간 오답 하이라이트**: 숫자를 입력하는 즉시 수도쿠 규칙 위반 여부를 검증하여 빨간색으로 표시.
- **커스텀 모달**: 시스템 `alert` 대신 웹 뷰 형태의 깔끔한 디자인 팝업 사용.
- **힌트 기능**: 현재 칸에 들어갈 수 있는 유효한 숫자 후보들을 작은 텍스트로 표시.

### 2.3. 데이터 관리 및 성능
- **상태 유지 (Persistence)**: `localStorage`를 활용하여 진행 중인 게임 상태(배열, 모드, 번호, 난이도 등)를 브라우저 캐시에 저장. 페이지 새로고침 시에도 기존 상태 복구.
- **성능 최적화**: 
    - 셔플 및 검증 로직을 O(N²) 수준으로 최적화하여 9x9 기준 0.01ms 이하의 연산 속도 확보.
    - 수백 개의 배열 데이터를 직접 저장하는 대신 알고리즘으로 생성하여 메모리 사용량 최소화.

## 3. 설치 및 실행 방법
1. 프로젝트 루트의 `sudoku/index.html` 파일을 웹 브라우저에서 실행.
2. (선택) `lib/easeljs-0.8.2.min.js` 파일이 상위 디렉토리에 존재해야 함.

## 4. 향후 개선 사항
- 힌트 사용 횟수 제한 기능.
- 게임 클리어 시간 기록(Timer) 및 베스트 레코드 저장.
- Backtracking 알고리즘을 통한 완전 자동 퍼즐 생성기 도입.
