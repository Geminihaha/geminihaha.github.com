# 핵심 알고리즘 명세서 (Core Algorithm Specs)

## 1. 백트래킹 기반 보드 생성 (Generator Engine)
1. 9x9 행렬을 0으로 초기화한다.
2. 1부터 9까지의 숫자를 무작위로 셔플(Shuffle)한 배열을 준비한다.
3. 순차적으로 셀을 방문하며 스도쿠 규칙(행, 열, 3x3 박스 중복 검사)을 통과하는 숫자를 삽입한다.
4. 만약 특정 셀에서 유효한 숫자가 없다면, 이전 단계로 돌아가 다른 숫자를 대입하는 **Backtracking(되추적)** 알고리즘을 수행하여 정답 보드(`solutionBoard`)를 완성한다.

## 2. 유일해 보장 마스킹 알고리즘 (Unique Solution Masking)
1. 완성된 보드에서 무작위로 한 셀을 골라 숫자를 제거(0으로 설정)한다.
2. 숫자가 제거된 보드를 스도쿠 솔버(`solver.js`)에 입력한다.
3. **솔버 로직 제한:** 솔버가 정답을 찾을 때, 첫 번째 정답을 찾은 후 탐색을 종료하지 않고 다른 가능성이 있는지 끝까지 탐색하도록 설계한다.
4. 정답 카운트가 2 이상이 되는 순간, 유일해 규칙이 깨진 것이므로 방금 지운 숫자를 다시 복원하고 다른 셀을 대상으로 마스킹을 시도한다.
5. 설정한 난이도 타겟 빈칸 개수(예: 중급 50개)에 도달할 때까지 위 과정을 반복한다.
